## Affected Pages/Routes
- Main launch window — 从资源管理器右键打开单个 `build.xml`，确认能识别项目、列出 target、启动/取消构建并展示实时输出。
- Control Center main window — 管理多个 `build.xml`，确认分组、批量导入、拖拽排序、单个构建、批量构建和状态统计都可用。
- Installer / uninstall flow — 安装、卸载、状态检查必须正确处理管理员权限、注册表写入和恢复。
- First-run desktop integration — 首次启动中控中心时，确认单实例唤起、桌面快捷方式创建和图标资源加载正常。

## Key Interactions to Verify
- 在资源管理器中右键一个有效 `build.xml`，主窗口能正常打开并默认选中“使用默认目标”。
- 在主窗口选择显式 target 并运行，实时输出持续刷新，成功后状态栏和日志目录行为正确。
- 在构建进行中点击取消，进程被终止，界面恢复可操作状态，用户能看到明确取消结果。
- 在中控中心中添加单个文件、整个文件夹和粘贴路径导入，重复文件被跳过，非 Ant XML 被拦截。
- 在中控中心中拖拽一个或多个文件到其他分组或组内新位置，顺序和分组持久化正确。
- 对一个分组执行批量构建，逐个输出结果，结束后统计和文件状态正确更新。
- 第二次启动中控中心时，已有实例被激活而不是打开第二个实例。
- 运行安装和卸载流程后，右键菜单项出现/消失与实际注册表状态一致。

## Edge Cases
- 传入不存在的文件路径、非 XML 文件或可疑 URI 参数时，用户看到明确报错而不是静默失败。
- `build.xml` 语法错误、无 target、或不是 Ant 项目时，主窗口和导入流程都能给出清晰提示。
- Java 或 Ant 未配置时，单构建和批量构建都能在开始前阻止执行并提示修复方向。
- 批量构建中某个文件丢失或失败时，其余文件的处理策略和最终汇总应一致且可理解。
- 首次运行无法创建桌面快捷方式时，不应阻塞主功能启动。
- 构建输出包含中文路径或非 UTF-8 编码时，输出区不能乱码或崩溃。

## Critical Paths
- Explorer right-click -> Main window -> Parse targets -> Run target -> Stream output -> Finish/cancel cleanly.
- Control Center launch -> Restore workspace -> Select file -> Run single build -> Persist last status.
- Control Center launch -> Select group -> Batch build -> Per-file result updates -> Final summary.
- Install -> Register context menu and launcher -> Right-click invocation works end-to-end.
- Uninstall -> Remove context menu and protocol stub -> No stale shell integration remains.
